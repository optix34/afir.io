📡 M25 Monitor — PILOT Extension

Расширение для системы мониторинга транспорта PILOT GPS (Pilot.Online / PILOT).

Основная задача: отображать в левой навигации только те транспортные средства, которые оснащены трекерами модели M25, и при выборе объекта загружать внешнюю панель https://mega-info.su/dealer2/ с информацией по выбранному автомобилю.

✅ Соответствие AI_SPECS.md
Расширение разработано в точности по контракту PILOT Extensions: не является SPA, не загружает Ext JS повторно, не содержит скриптов в doc/index.html, использует глобальный skeleton и прокси /store/....

📦 Состав расширения

m25_monitor/
├── index.html               # корневой файл для статического хостинга (пустой/перенаправление)
├── Module.js                # точка входа, класс Store.m25_monitor.Module
├── doc/
│   └── index.html           # эта документация
└── view/
    ├── Navigation.js        # левая панель (дерево с фильтрацией M25)
    └── MainPanel.js         # правая панель (iframe + тулбар)

⚙️ Как это работает

  1. Инициализация – после загрузки PILOT вызывает initModule() класса Store.m25_monitor.Module.
  2. Создание интерфейса – создаётся вкладка Pilot.utils.LeftBarPanel с деревом из Navigation.js и правая панель MainPanel.js, они связываются через navTab.map_frame = mainPanel.
  3. Загрузка данных – Navigation.js запрашивает /ax/tree.php?vehs=1&state=1.
  4. Фильтрация – рекурсивно обходит дерево, ищет узлы с типом «транспорт» (veh или vehid) и проверяет наличие подстроки m25 в поле оборудования. Пробует следующие имена полей: equipment, model, device, hardware, devicetype, tracker. Если ни одно не найдено – ищет в любом строковом поле, содержащем «equip» или «device».
  5. Отображение дерева – в левой панели показываются только папки, содержащие M25-объекты, и сами объекты. Для каждого объекта отображаются: название, IMEI, модель оборудования.
  6. Выбор объекта – при клике на транспорт в правой панели загружается https://mega-info.su/dealer2/?vehicle_id=XXX (или просто базовый URL, если ID не задан).
  7. Правая панель – содержит iframe с внешней страницей и тулбар: кнопки «Обновить», «Открыть в новом окне», строка с названием выбранного объекта.

🔧 Установка и регистрация

Требования к хостингу

  • Любой статический хостинг (Cloudflare Pages, GitHub Pages, VPS с Nginx).
  • Файлы должны быть доступны по публичному URL.
  • Корневая папка должна содержать Module.js и doc/index.html, а также подпапку view/ с файлами Navigation.js и MainPanel.js.
  • Для Cloudflare Pages обязателен файл index.html в корне (может быть пустым или копией doc/index.html).

Пошаговая регистрация в PILOT

  1. Загрузите все файлы расширения на ваш хостинг (например, Cloudflare Pages).
  2. Проверьте, что URL https://ваш-хост/Module.js открывается и показывает JavaScript-код.
  3. Войдите в PILOT как администратор.
  4. Перейдите в Администрирование → Приложения → Список расширений (или просто «Приложения»).
  5. Нажмите «Добавить».
  6. Заполните поля:
    • Название (slug): m25_monitor (только латиница, строчные буквы, подчёркивание). Не используйте дефисы.
    • URL: https://ваш-хост/ (обязательно с завершающим слешем).
  7. Сохраните расширение и включите его для нужного пользователя или для всех.
  8. Полностью перезагрузите PILOT (Ctrl+F5, чтобы обновить Ext JS приложение).
  9. В верхней панели или в левом меню (зависит от версии) должна появиться иконка M25 Monitor.
⚠️ Важное замечание про iframe
Внешний сайт https://mega-info.su/dealer2/ может запрещать встраивание через заголовок X-Frame-Options или CSP. Если в консоли браузера появится ошибка Refused to display in a frame, используйте кнопку «Открыть в новом окне» на тулбаре правой панели – она будет работать всегда.

🩺 Диагностика (если объекты не отображаются)

1. Проверьте консоль браузера (F12)

При успешной загрузке вы должны увидеть:

[M25] Extension initialized successfully
[M25] Raw tree data sample: { ... }
[M25] Filtered vehicles count: X

Если сообщений нет – расширение не зарегистрировано или Module.js не загрузился.

2. Проверьте фильтрацию

  • Откройте вкладку Network, найдите запрос к /ax/tree.php?vehs=1&state=1. Посмотрите ответ (Preview).
  • Найдите первый объект-транспорт и проверьте, в каком поле хранится модель трекера (например, equipment, model, device).
  • Если поле называется иначе, отредактируйте массив candidates в методе extractEquipment файла view/Navigation.js.
  • Если поле содержит не «M25», а, скажем, «M25L» или «M25 PRO», фильтр сработает, т.к. используется indexOf('m25') – он найдёт подстроку.

3. Проверьте связь map_frame

В консоли выполните:

var nav = skeleton.navigation.items.findBy(c => c.title === 'M25 Monitor'); console.log(nav.map_frame);

Должна отобразиться правая панель (Store.m25_monitor.view.MainPanel). Если undefined – проблема в модуле.

🔄 Обслуживание и обновления

Расширение полностью автономно. Для обновления достаточно заменить файлы на хостинге и очистить кэш браузера (Ctrl+F5). PILOT автоматически подхватит новую версию, так как Module.js загружается при каждой перезагрузке интерфейса.

Изменения вступают в силу после полной перезагрузки страницы PILOT.
